GenAI Assessment Recommendation System — Approach Document

Author: Devdatta Patil  |  Date: April 2026  |  GitHub: github.com/devdattapatilll/SHL-assessment1

1. Problem Statement

Build a web-based Retrieval-Augmented Generation (RAG) tool that accepts natural language queries or job descriptions and recommends the most relevant SHL assessments from their product catalog. The system must maximize Mean Recall@10 against a provided evaluation dataset.

2. Architecture Overview

Pipeline: Query → Preprocessing → Hybrid Search (FAISS + BM25) → Reciprocal Rank Fusion → K/P Balancing → Top-10 Results

2.1 Data Collection & Processing

  • Web Scraper: Built a BeautifulSoup-based scraper targeting SHL's product catalog across 32 paginated pages, extracting 389 unique Individual Test Solutions.
  • Fields Extracted: Name, URL, description, duration (parsed to integer minutes), adaptive/IRT support, remote testing support, and test type codes (mapped to descriptive labels: Knowledge & Skills, Personality & Behavior, etc.).
  • Normalization: Emoji indicators converted to Yes/No strings, test type codes (K, P, A, B, etc.) mapped to human-readable categories, duplicate URLs de-duplicated.

2.2 Embedding & Indexing

  • Model: all-MiniLM-L6-v2 from Sentence-Transformers (384-dim embeddings, optimized for semantic similarity).
  • Document Representation: Each assessment is embedded as: name | description | test_type — combining identity, content, and category for rich semantic coverage.
  • Vector Store: FAISS IndexFlatIP with L2-normalized vectors, enabling cosine similarity via inner product. Persistent storage to disk (faiss_index.bin).

2.3 Retrieval Pipeline (Hybrid RAG)

StageMethodPurpose
1. Dense RetrievalFAISS (Top-20)Semantic similarity for conceptual matching
2. Sparse RetrievalBM25/Okapi (Top-20)Keyword matching for exact term relevance
3. FusionReciprocal Rank Fusion (k=60)Combine dense + sparse with weighted RRF scores
4. Re-rankingWeighted score aggregationSemantic (70%) + BM25 (30%) fusion weights
5. BalancingK/P type allocationEnsure mix: ~50% Knowledge, ~30% Personality, ~20% Other

Why Hybrid? Pure vector search misses exact keyword matches (e.g., "Java", "SQL"), while pure BM25 misses semantic intent (e.g., "developer" → "programmer"). RRF fusion captures both.

2.4 API Layer

  • Framework: FastAPI with Pydantic validation
  • Endpoints: GET /health (status check), POST /recommend (accepts JSON {"query": "..."}, returns {"recommended_assessments": [...]})
  • Optional LLM: Google Gemini 2.0 Flash for query understanding and skill extraction (gracefully degrades if unavailable)
  • Deployment: Render (free tier web service)

3. Frontend

  • Framework: Streamlit with custom CSS (dark theme, gradient cards, responsive grid)
  • Features: Text/JD input, ranked results with test-type badges, duration/remote/adaptive metadata, direct links to SHL catalog
  • Deployment: Streamlit Community Cloud (self-contained — embeds the pipeline directly, no external API dependency)

4. Evaluation Methodology

4.1 Dataset

Provided Excel file with 10 unique queries and 65 ground-truth query-URL pairs (5-10 relevant assessments per query).

4.2 Metric: Mean Recall@10

Recall@10 = |relevant assessments in top 10 predictions| / |total relevant assessments|
Mean Recall@10 = average(Recall@10 across all queries)

4.3 URL Normalization Challenge

Ground truth URLs use inconsistent prefixes (/solutions/products/ vs /products/) and trailing slashes. Solution: Compare by URL slug only (final path segment), making matching robust to format variations.

4.4 Results

ConfigurationMean Recall@10Notes
Baseline (FAISS only)0.02Pure semantic search, no keyword matching
Hybrid (FAISS + BM25 + RRF)0.178.5× improvement over baseline

Key Insight: The dramatic improvement comes from BM25 capturing exact technology/skill keywords (Java, SQL, Excel) that semantic embeddings alone generalize too broadly.

6. Future Improvements

  • Cross-encoder re-ranking (e.g., ms-marco-MiniLM-L-6-v2) for finer-grained relevance scoring
  • Query expansion using LLM-generated synonyms and related skills
  • Duration/constraint filtering as hard post-retrieval filters
  • Fine-tuned embeddings on HR/assessment domain data
  • Larger evaluation set to validate generalization
